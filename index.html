About

I'm a security practitioner in the making — someone who learns by doing, not just studying. My approach is simple: build the lab, run the attack, read the logs, write it up.

I got interested in cloud security because that's where the real attacks happen now. Most breaches aren't movie-style intrusions — they're a misconfigured S3 bucket, a forgotten access key, a developer account with one too many permissions.

I want to be the person who catches those. So I practice finding them first.

Currently preparing for cloud security and SOC analyst roles, with hands-on labs in AWS attack simulation, log analysis, and incident response.
